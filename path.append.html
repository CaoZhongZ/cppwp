30.10.28.4.3 path appends [path.append]

1
#

The append operations use operator/= to denote their semantic effect of appending preferred-separator when needed.

path& operator/=(const path& p);

2
#

Requires: !p.has_­root_­name().

3
#

Effects: Appends path​::​preferred_­separator to pathstring unless:

  • (3.1)

    an added directory-separator would be redundant, or

  • (3.2)

    an added directory-separator would change a relative path into an absolute path [ Note: An empty path is relative. â€” end note ] , or

  • (3.3)

    p.empty() is true, or

  • (3.4)

    *p.native().cbegin() is a directory-separator.

Then appends p.native() to pathstring.

4
#

Returns: *this.
